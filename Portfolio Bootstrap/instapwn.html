Abstract

It has always been a tedious process to download any kind of content from Instagram due to lack of a download feature. There are some third party solutions that claim to make the download process smoother. But those come with annoyances such as popup advertisements, data collection policies ,etc. On the other hand Instagram's API can also be used for the same purpose but API are generally given lower priority compared to users. Due to this API can stop providing data in times of heavy traffic on the website. While designing a solution to solve this issue with users privacy and user experience as first. For that we had to reverse engineer how data is received on the client side and store it in bulk.

Introduction

Instapwn is a Data Scraping Utility that focuses on making the data extraction process for Instagram simpler. It is also capable of downloading content from private accounts as long as you follow the account (Login Required). It's pronounced as “Insta-own” even if its spelled as “Instapwn”.

Project Structure

The following are important files/directories in the project. launcher.py: Launches chromedriver and the Instapwn ui.ui file. Also contains all the logic responsible for all the features offered by Instapwn chromedriver/: Contains chromedriver.exe (selenium webdriver) which enables chrome to be controlled by external scripts. Note: Google Chrome should be pre installed on the system. In case the chromedriver is outdated it can be updated from here (Stable Release Recommended). webAssets/: Contain webassets to be rendered in chromedriver to indicate completion of download process, etc. source_rc.py: Contains .png assets converted into one single python file. PyQt5 autogenerates this using source.qrc file. ui.ui: Contains xml code for the Instapwn UI. This can be converted to .py to improve performance but we decided to use .ui file since working with .xml is convenient if quick changes had to be made. saved/: This is where we store extracted multimedia content.

How does it work?

We had observed that Instagram generates HTML on the client side using some variant of Js. This was a major issue since working with Js and Beautiful soup would've been way more difficult. Then we decided to use Selenium (chromedriver) to render HTML to make our workflow easier. This gave us another benefit of not dealing with user logins since users can easily use the official login method in the web browser right away. Instapwn code consists of following features: Bulk Images Download: This feature works by taking the username of the target user as input. Once their instagram page is loaded we store links of all the loaded images in a list. Then the program iterates through all the elements of the list until all the images are downloaded. Note: Since instagram uses Ajax to load everything. Due to this currently supports upto 12(sometimes 24 depending on how the page has been loaded). Single Image Download: Capable of downloading specific images using direct link. Stories Download: Can download both Image based and Video Stories. Designed to work without any input. So its possible to use Story downloader by just opening a story and clicking the desired extraction button. IGTV Videos Download: Regular/IGTV videos are very similar to how images can be extracted. All we need to do is find out the video tag and get its src attribute. Multi Images/Videos Download: This feature calculates the number of Multi-Images(or videos) contained in a single post (Instagram limits 10 as maximum). The link to post isn't totally necessary but can be provided optionally. Finally the index of the desired Multi-Images(or videos) is needed to fetch the right index of <img> tag. Additionally since instagram uses Ajax an additional step to manually load the desired image/video in ChromeDriver. Feedback: As an app developer feedback/bug reports are very essential. So we made sure to add a feedback button which can be used to directly contact us developers. Its basically a mailto link that will launch your default mail client. There is also a fair share of file handling and string manipulation involved in the making of this project. And requests library is used for downloading content using extracted links.
